# Minimal Evidence UI Implementation Plan

## Purpose

This plan maps the existing PathWarden desktop shell to the v0.1.7 Evidence UI Shell goal.

The current desktop app already exists under:

```text
apps/desktop

Current structure:

apps/desktop/main/main.cjs
apps/desktop/preload/preload.cjs
apps/desktop/ui/src/index.html
apps/desktop/ui/src/renderer.js
apps/desktop/ui/src/styles.css
apps/desktop/package.json

The first UI implementation should use this existing shell instead of introducing React, Vite, or a new frontend structure.

Current baseline
pw-v0.1.6-release-evidence-and-doc-cleanup
v0.1.7 target
pw-v0.1.7-evidence-ui-shell
Build posture
use existing Electron shell
plain HTML/CSS/JS first
read-only UI first
no React yet
no Vite yet
no TypeScript UI migration yet
no Rust
no federation runtime
no execution controls
Why this approach

The current shell is small and already uses Electron.

Adding React/Vite now would introduce toolchain complexity before the UI contract is proven.

The first goal is only to show PathWarden's evidence posture clearly.

Allowed first-pass UI behavior

The UI may:

display latest report index
display governance report status
display replay provenance status
display federation readiness status
display release_safe
display admissible
display lineage_complete
display ready_for_federation
display local report paths
display missing report state

The UI must not:

generate reports
verify reports
run diagnostics
execute actions
approve actions
deny actions
modify reports
modify policies
modify authority records
trigger federation
sign artifacts
call networks
Data source

First-pass UI should read:

exports/report-index/latest-report-index.json

This index is generated by:

npm run export:latest-report-index

The UI should not scan arbitrary folders in the first pass.

Required precondition

Before launching the UI, the user should generate current evidence artifacts:

npm run export:governance-report
npm run export:replay-provenance-report
npm run export:federation-readiness-audit
npm run export:latest-report-index
npm run verify:latest-report-index
Implementation slices
Slice 1: Inspect current shell files

Read:

apps/desktop/main/main.cjs
apps/desktop/preload/preload.cjs
apps/desktop/ui/src/index.html
apps/desktop/ui/src/renderer.js
apps/desktop/ui/src/styles.css

No edits.

Slice 2: Add read-only preload API

Expose one read-only function:

window.pathwardenEvidence.readLatestReportIndex()

This function should:

read exports/report-index/latest-report-index.json
return parsed JSON
return a safe missing-file result if the index does not exist
not write files
not execute commands
not scan arbitrary directories
Slice 3: Update renderer

Renderer should:

call readLatestReportIndex()
display evidence overview cards
display missing index state if needed
display local report paths
show explicit non-runtime boundary
Slice 4: Update HTML/CSS

HTML/CSS should provide:

Evidence Overview
Governance Report card
Replay Provenance card
Federation Readiness card
Boundary notice
Artifact paths
Slice 5: Add desktop smoke script if useful

Optional package script in root or app package later:

desktop:start

Only if needed.

First screen layout

Recommended simple layout:

Header:
PathWarden Evidence Shell

Banner:
Read-only evidence viewer. No actions are executed from this UI.

Cards:
Governance Report
Replay Provenance
Federation Readiness

Footer:
Federation readiness is advisory and non-executable.
Status text rules

Governance:

release_safe true = Release-safe
release_safe false = Not release-safe
null = No governance report available

Replay provenance:

admissible true = Admissible
admissible false = Not admissible
lineage_complete false = Lineage incomplete
null = No replay provenance report available

Federation readiness:

ready_for_federation true = Federation design may be considered
ready_for_federation false = Federation not ready
null = No federation readiness audit available
Missing index behavior

If index is missing, show:

No latest report index found.
Run:
npm run export:governance-report
npm run export:replay-provenance-report
npm run export:federation-readiness-audit
npm run export:latest-report-index
Testing

Minimum verification after implementation:

npm run check
npm run diag
npm run export:latest-report-index
npm run verify:latest-report-index
git status --short

Desktop manual check:

cd apps/desktop
npm start

Expected manual result:

desktop app launches
evidence overview displays latest report index
no execution controls are visible
federation not-ready/advisory boundary is visible
Future upgrade path

Only consider React/Vite after the plain shell proves:

the data contract is correct
the evidence view is useful
the layout needs component structure
manual UI maintenance becomes awkward
Summary

Use the existing Electron shell.

Do not introduce a new frontend stack yet.

The goal is read-only evidence visibility, not UI architecture expansion.
